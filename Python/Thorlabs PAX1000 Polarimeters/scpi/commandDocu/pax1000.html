This is the description of the Thorlabs PAX1000 remote interface. This polarimeter can be remotely controlled using the SCPI (Standard Commands for Programmable Instruments) interface.

In 1975, the IEEE standardized a bus developed by Hewlett-Packard originally called HPIB (Hewlett-Packard Interface Bus), later changed to GPIB (General Purpose Interface Bus). The standard was called IEEE 488 (IEEE 488.1) and it defined mechanical aspects of the bus. The later IEEE 488.2 defined its protocol properties. What was missing were set of rules between the manufacturers on commands to control the instruments. Sometimes these varied even between different models from the same manufacturer.

In 1990 SCPI Consortium released the first SCPI standard as an additional layer for the IEEE-488.2 standard.

SCPI commands are ASCII strings, which are sent to instrument over the physical communication layer. They can perform:

  • Set operations, for example the *RST command (resetting the instrument).
  • Query operations, for example the *IDN? query (querying the instrument’s identification string).

Some SCPI commands exist as both set commands and query commands. An example is a polarimeters's wavelength command SENS:CORR:WAV. You can set it with the SCPI command SENS:CORR:WAV 832, and also query its current value with the SENS:CORR:WAV?.

The format mentioned in this manual e.g.: SENSe#:CALCulate[:MODe] <pres> is called canonical form. Here are the most important rules to remember:

  • The parts within square brackets are not mandatory and can be left out.
  • All # within the command represent a suffix positive integer number starting at 1. If there is only one suffix you may skip it because default is applied internally. If there are two or more suffix it is mandatory to specify all.
  • The capital letter parts are mandatory; the small letters can be omitted. This is called short form. An example of the above command in the short form is SENS:CALC 1. You can use either the short form, or the long form SENS:CALCulate 1, but nothing in between, e.g. SENS:CALCul 1.
  • The SCPI commands are case-insensitive. You can also use the short form sens:calc 1 or or the long form sens:calculate 3
  • Combine multiple commands into one string using selmicolon ';'. For example, a combined string of SENS:CALC 1 and SENS:CORR:WAV 850 is SENS:CALC 1;CORR:WAV 850. Notice that the second command does not have the SENS: part. The reason is, that the command tree path does not change within one string. If you want to reset the command tree path to the root, use the colon character at the beginning of the second command: SENS:CALC 1;:SENS:CORR:WAV 850.
  • Create query forms by adding a question mark, mostly to the end: SENS:CORR:WAV? Sometimes there is an additional parameter placed after the question mark. There must be a space character between the question mark and the additional parameter. For example: SENS:CORR:WAV? MIN

The complete SCPI standard is available here: SCPI-99

:DATA[:PRIMary][:PULL]:OLDest:BINary?"
Suffixe

Response
  • lengthbinary unsigned half word with length of binary measurement data
  • binMeasRes1binary optional binary measurement data. Only available if length is not zero
  • binMeasRes2binary optional binary measurement data. Only avaiable if length is larger than 52.
Use this command to pull the latests avaiable measurement results out of the internal result queue in binary format. The query will return 1 or 2 measruement results to keep the queue empty in continous measurements. The command has to be used for fast measurements to access all data. The internal queue will buffer the recent 3 measurement results. If you query to slow the device will overwrite the oldest elements with new data. This results in a loss of sample data. You can use evulution count or timestamp in the results to detect loss of data. In case the queue is currenlty empty no data is returned. The return data equals the data of command SENS#:DATA:LAT? bin binary encoding.
Integers and IEEE compliant flaots are all 4 Byte with little endian byte order. To parse the data read 2 bytes first. The unsigned half word contains the length of the binary response. If the two bytes length is 0 there is no data to query. If the results is 52 it contains on result. If the length is 104 it contains two results.

The following python code should give you an example how to parse the binary response.
def parseBinPax1000Data(bin:bytearray):
   dataSet = []
   i = 0
   dataSet.append(struct.unpack('<I', bytearray(bin[i:i+4]))[0]) #revCount
   i += 4
   dataSet.append(struct.unpack('<I', bytearray(bin[i:i+4]))[0]) #timeStamp
   i += 4
   dataSet.append(struct.unpack('<I', bytearray(bin[i:i+4]))[0]) #PAXopmode
   i += 4
   dataSet.append(struct.unpack('<I', bytearray(bin[i:i+4]))[0]) #statusFlags
   i += 4
   dataSet.append(struct.unpack('<I', bytearray(bin[i:i+4]))[0]) #gainIdx
   i += 4
   dataSet.append(struct.unpack('<I', bytearray(bin[i:i+4]))[0]) #adcMin
   i += 4
   dataSet.append(struct.unpack('<I', bytearray(bin[i:i+4]))[0]) #adcMax
   i += 4
   dataSet.append(struct.unpack('<f', bytearray(bin[i:i+4]))[0]) #revTime
   i += 4
   dataSet.append(struct.unpack('<f', bytearray(bin[i:i+4]))[0]) #misAdj
   i += 4
   dataSet.append(struct.unpack('<f', bytearray(bin[i:i+4]))[0]) #theta
   i += 4
   dataSet.append(struct.unpack('<f', bytearray(bin[i:i+4]))[0]) #eta
   i += 4
   dataSet.append(struct.unpack('<f', bytearray(bin[i:i+4]))[0]) #DOP
   i += 4
   dataSet.append(struct.unpack('<f', bytearray(bin[i:i+4]))[0]) #Ptotal
   return dataSet

STAT:AUX?Queries the SCPI Auxiliary Event register
STATus:AUXiliary[:EVENt]?
Response
  • bitmaskuint For the description of the bits see function
The auxiliary event registers captures changes in the auxiliary condition register. Each event bit corresponds to the same bit in the condition register. An event bit is set to 1 when the associated condition bit changes from 0 to 1. Once set the bits are sticky. That is, they will not be reset even if the condition bit resets to 0 again. Event register bits will only be reset to 0 by reading the event register in the application. This guarantees that the application cannot miss an event that occurs between the reading and clearing of the event register.

SYST:ERR?Reads and removes oldest element of SCPI error queue
SYSTem:ERRor[:NEXT]?
Response
  • errorint SPCI error code. If no error occurred result will be 0.
  • errDescstring error code English description text.
Call to test if there where errors during command execution or asynchronous device errors. The device internally uses a error queue to store SCPI errors. This command returns and removes the oldest element of the queue. Call multiple times until queue is empty and function returns 0, "No error". The device stores errors in a queue containing up to 10 entries. So if you do not keep the queue empty you might loose errors when queue is full.
